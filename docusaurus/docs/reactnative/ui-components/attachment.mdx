The `Attachment` is the UI component for rendering attachments for message, within MessageList. This component delegates rendering part to following sub-components, as per type of attachment:

- Card
- FileAttachment
- Gallery
- Giphy
- UrlPreview

:::note
This component is used within library to render all the attachments EXCEPT type `image`. For images, [`Gallery`](../gallery) component is used.
:::

You can set `Attachment` prop on `Channel` component to replace this default component:

```tsx
<Channel Attachment={CustomAttachment} />
```

:::caution
Generally you wouldn't need to use this component for customization, you should instead use specific attachment type related prop such as Giphy, Gallery, UrlPreview etc.
Please check our guide on [Attachment Customization](../guides/attachment_customizations.mdx) for details.
:::

## Props

### <div class="label required">required</div> **attachment**

Attachment object to render UI for. This is basically a value of `attachments` property on message object - `message.attachments[0]`

```
{
  actions?: Action[];
  asset_url?: string;
  author_icon?: string;
  author_link?: string;
  author_name?: string;
  color?: string;
  fallback?: string;
  fields?: Field[];
  footer?: string;
  footer_icon?: string;
  image_url?: string;
  og_scrape_url?: string;
  pretext?: string;
  text?: string;
  thumb_url?: string;
  title?: string;
  title_link?: string;
  type?: string;
}
```
